# goit-algo-hw-04

## Завдання 1 — Рекурсивне копіювання та сортування файлів за розширенням

Скрипт `modul1.py` рекурсивно обходить вихідну директорію, копіює всі файли до директорії призначення та розподіляє їх по піддиректоріях за розширенням (`.txt`, `.jpg`, `.py` тощо).

---

## Вимоги

- Python 3.6 або новіший

---

## Швидкий старт (для перевірки)

Скопіюйте і вставте один рядок у термінал — він створить тестові файли, запустить скрипт і покаже результат:

```bash
mkdir test_folder\subdir; echo text > test_folder\a.txt; echo x > test_folder\b.jpg; echo x > test_folder\subdir\c.csv; python modul1.py test_folder; tree dist /F
```

---

## Запуск

### Синтаксис

```bash
python modul1.py <вихідна_директорія> [директорія_призначення]
```

- `<вихідна_директорія>` — обов'язковий аргумент, шлях до папки з файлами
- `[директорія_призначення]` — необов'язковий, за замовчуванням створюється папка `dist`

---

## Покрокова інструкція для перевірки

### Крок 1 — Клонувати або завантажити репозиторій

```bash
git clone https://github.com/Ievgen-tech/goit-algo-hw-04.git
cd goit-algo-hw-04
```

### Крок 2 — Створити тестову папку з файлами

```bash
mkdir test_folder
mkdir test_folder\subdir
echo "hello world" > test_folder\file1.txt
echo "some text" > test_folder\notes.txt
echo "print('hi')" > test_folder\script.py
echo "" > test_folder\image.jpg
echo "data" > test_folder\subdir\nested.csv
```

### Крок 3 — Запустити скрипт

> Переконайтесь, що Крок 2 виконано — папка `test_folder` має існувати до запуску.

**Варіант А** — destination за замовчуванням (`dist`):

```bash
python modul1.py test_folder
```

**Варіант Б** — вказати власну папку призначення:

```bash
python modul1.py test_folder my_output
```

### Крок 4 — Перевірити результат

```bash
tree dist /F
```

Очікувана структура:

```text
dist/
├── txt/
│   ├── file1.txt
│   └── notes.txt
├── py/
│   └── script.py
├── jpg/
│   └── image.jpg
└── csv/
    └── nested.csv
```

## Завдання 2 — Фрактал «Сніжинка Коха»

Скрипт `modul2.py` використовує рекурсію для побудови та візуалізації фракталу «Сніжинка Коха».
Користувач вказує рівень рекурсії — що більший рівень, то детальніша сніжинка.

### Вимоги для запуску

- Python 3.6 або новіший
- Модуль `turtle` (вбудований у стандартну бібліотеку Python, встановлення не потрібне)

### Як запустити

```bash
python modul2.py
```

Після запуску програма запитає рівень рекурсії у консолі:

```text
=== Сніжинка Коха ===
Введіть рівень рекурсії (рекомендовано 1–6):
```

Введіть ціле число і натисніть Enter — відкриється графічне вікно зі сніжинкою.

### Рівні рекурсії

| Рівень | Результат |
| ------ | --------- |
| 0 | Простий трикутник |
| 1 | Зірка з 6 кутів |
| 2–3 | Початкова форма сніжинки |
| 4–5 | Деталізована сніжинка |
| 6+ | Дуже детально, малюється довше |

### Швидка перевірка

Запустити та ввести `4` → натиснути Enter → відкриється вікно з детальною сніжинкою.

```bash
python modul2.py
```

## Завдання 3 — Порівняльний аналіз алгоритмів сортування

Скрипт `modul3.py` порівнює три алгоритми сортування — **Insertion Sort**, **Merge Sort** та **Timsort**
(вбудований Python) — за часом виконання на різних типах і розмірах масивів.

### Як запустити modul3

```bash
python modul3.py
```

Програма автоматично виконає тести і виведе таблицю результатів у консоль.

### Результати вимірювань (timeit, 5 повторень)

#### Випадковий масив

| Розмір | Insertion Sort | Merge Sort | Timsort |
| ------ | -------------- | ---------- | ------- |
| 100 | 0.000109 s | 0.000076 s | 0.000003 s |
| 1 000 | 0.011286 s | 0.001065 s | 0.000068 s |
| 5 000 | 0.289162 s | 0.006072 s | 0.000358 s |
| 10 000 | — | 0.015505 s | 0.000824 s |

#### Майже впорядкований масив

| Розмір | Insertion Sort | Merge Sort | Timsort |
| ------ | -------------- | ---------- | ------- |
| 100 | 0.000023 s | 0.000058 s | 0.000001 s |
| 1 000 | 0.001910 s | 0.000742 s | 0.000009 s |
| 5 000 | 0.049734 s | 0.004268 s | 0.000050 s |
| 10 000 | — | 0.009201 s | 0.000117 s |

#### Зворотньо відсортований масив

| Розмір | Insertion Sort | Merge Sort | Timsort |
| ------ | -------------- | ---------- | ------- |
| 100 | 0.000206 s | 0.000059 s | 0.000001 s |
| 1 000 | 0.022011 s | 0.000682 s | 0.000004 s |
| 5 000 | 0.572133 s | 0.004128 s | 0.000019 s |
| 10 000 | — | 0.008433 s | 0.000040 s |

> `—` — пропущено через надто великий час виконання O(n²)

### Висновки

1. **Timsort є найшвидшим** на всіх типах даних і розмірах масивів.
   На масиві з 10 000 елементів він у ~19 разів швидший за Merge Sort
   і у сотні разів швидший за Insertion Sort.

2. **Timsort ефективно використовує впорядкованість даних.**
   На майже впорядкованому масиві з 10 000 елементів він у ~79 разів
   швидший за Merge Sort (0.000117 s проти 0.009201 s). Це пояснюється
   тим, що Timsort виявляє вже впорядковані підпослідовності (runs) і
   обробляє їх без зайвих порівнянь.

3. **Merge Sort стабільний O(n log n)**, але має накладні витрати через
   рекурсію та постійне виділення нових списків у пам'яті. Timsort
   уникає цього, використовуючи Insertion Sort для малих runs (~32–64
   елементи) і злиття лише там, де це необхідно.

4. **Insertion Sort доцільний лише** для дуже малих масивів (до ~100
   елементів) або майже впорядкованих даних. На великих масивах його
   складність O(n²) робить його непрактичним: 5 000 елементів —
   0.29 s проти 0.000358 s у Timsort (у ~800 разів повільніше).

5. **Практичний висновок:** вбудовані функції `sorted()` та `.sort()`
   використовують Timsort і є оптимальним вибором для переважної
   більшості задач. Реалізовувати власний алгоритм сортування на Python
   виправдано лише в навчальних або вузькоспеціалізованих цілях.
